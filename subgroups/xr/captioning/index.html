Our work on developing guidance for captioning in XR is using the following process that is described in detail within the content creation process for migrating WCAG 2.1 and will also conside WCAG 2.2 Success Criteria Acceptance Requirements :

  1. Define Captioning
  2. Identify User Needs
  3. Write Functional Outcomes
  4. Write the Tests
  5. Write Methods to Pass Tests
  6. Write how to and supporting documentation
  7. Write the Guidelines

0. Define Captioning

We have investigated how captioning and audio descriptions are defined across various sources. Information on this can be seen on the Defining Captioning in Mixed Reality page.

1. Identify User Needs

We have carried out a task that has taken XR Accessibility User Requirements and identified those that are relevant to captioning. Information on this can be seen on the Captioning XR User Requirements page.
